# Trading Framework — Notes & Documentation

All analysis, learnings, tasks, plans, and design specs live here.

## Structure

- **`analysis/`** — Point-in-time analyses of the framework (strengths, weaknesses, gaps). Filename: `YYYY-MM-DD-<topic>.md`
- **`learnings/`** — Lessons learned during development, "gotchas," insights from trades or research. Filename: `YYYY-MM-DD-<topic>.md`
- **`plans/`** — Future work, roadmaps, improvement backlogs. Filename: `<topic>.md` (living docs) or `YYYY-MM-DD-<topic>.md` (dated).
- **`specs/`** — Design documents for new features before implementation. Filename: `YYYY-MM-DD-<feature>-design.md`
- **`tasks/`** — Specific task-level notes or checklists for in-progress work. Filename: `YYYY-MM-DD-<task>.md`

## Conventions

- Date-prefix filenames when time-of-observation matters (analyses, learnings).
- Use `plans/IMPROVEMENTS.md` style for living documents that get edited over time.
- Keep each doc focused on one topic. Cross-link liberally.
